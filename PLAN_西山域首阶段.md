# 山海经 MOD — 仅西山域（首阶段验证）

## 范围约束

- **只做西山域**：世界规则、地图片段、生物与道具均以《西山经》意象与可读性强的原型为主。
- **本阶段不做**：南山 / 北山 / 东山 / 中山 的任何地图、叙事线、共享五域系统；也不在文档与配置里预留多域开关（避免分散注意力）。
- **技术前提**：目标为 **饥荒联机版（DST）** MOD；当前仓库仅有计划文档时，需从零建立 Klei 标准目录与入口（`modinfo.lua`、`modmain.lua` 等）。

## 空间形态（已定稿）

- **西山域出现在主大陆上**：在世界生成阶段划定 **一块连续地区**（自定义生物群系 / `task` / 布局与噪声配置），玩家 **跑图即可进入**，无需传送门、非独立 `shard`、非洞穴层或室内子世界。
- **不改原图原则**：西山域只在地图生成中的**空闲区域**落位，不覆盖主世界既有关键区域与默认结构，不做整图重写。
- **边界**：以地块与布景过渡为主（与相邻 `forest`/`savanna` 等可硬边或渐变）；首阶段不要求空气墙或强制传送，以「地图上可辨认的一片西山域」为准。

## 实现备忘（与 DST 机制对齐）

避免将「空闲区域、不改原图、局域天气」仅停留在产品语言，动工时须对应到具体技术手段：

- **「空闲区域」的工程含义**：DST 主世界由 `story` / `tasks` 等拼合版图。常见路径包括：**新增独立 task 参与默认 level 的生成**（由生成器在版图内落一块西山域），或 **生成结束后按规则改写地皮/布景**（须严格限定条件，否则易与「不覆盖原有关键内容」冲突）。实现前锁定一种主方案并写简短技术笔记，多 seed 验证与默认地图的邻接关系。
- **「关键区域」可验收清单**：动工前列出需刻意保护的对象（例如出生门附近半径、特定固定奇遇、`wormhole` 走廊等——按项目取舍），避免验收里「关键」一词无法对证。
- **第二阶段「天气」**：DST 多为**世界级**天气系统；若要做西山域独有体感，通常采用 **区域内雾、粒子、环境声、色彩校正、局部 ambient** 等组合表达「域内气候」，与「只改该域季节数据」区分开，以免预期落空。

## 西山域在玩法上的「可感知效果」（当前阶段验收）

当前阶段目标是先把“区域生成”跑通并稳定，建议满足以下验收：

1. **大陆上的专属地区**：新档生成后，地图上存在 **可定位的一块西山域**，并且在空闲区域生成，不替换原有关键群系/结构。
2. **基础可辨识**：地表贴图或布景层面能识别“这是西山域”，但不要求完整建筑、动植物和天气玩法。
3. **与主世界一体**：死亡、回档、客机同步均走主世界逻辑，不依赖跨 shard 或子场景加载。

建筑、动植物、天气将作为下一阶段内容，在区域生成稳定后再做详细设计与接入。

## 第二阶段方向：完全虚构神山 + 「眼前一亮」

气质定调为 **志怪神山**，而非写实古村落考据。建筑可以大胆「不像人间常态」，只要在符号与叙事上仍让人读出《山海经》——**荒远、神异、不可全解**。第二阶段优先做 **1～2 个神山级奇观地标** 撑住第一印象，再补次要构筑物。

建议把握以下原则（实现时再细化到具体 prefab 清单）：

- **虚构优先**：山体结构、悬空阶、穿石梁、晶脉外露、雾中剪影等可**超现实**，不必拘泥真实营造法式；目标是玩家进域后 **视线立刻被吸住**。
- **奇观地标**：至少一座「主神山」式构筑（或复合体）——体量显著大于主世界常规建筑，轮廓可读（远景也能认），可配合局部动态（光、雾、粒子、微弱动画）强化记忆点。
- **山海经符号锚点**：用金玉色泽、祭祀台、堆石阵、兽面/羽纹抽象装饰、经名化命名等**锚定文本联想**，与纯西式高塔/城堡剪影拉开距离。
- **色彩与光**：在木石基底上允许**更戏剧的光色对比**（例如冷雾中的暖金玉色、裂隙微光），服务于「惊艳感」，但仍保持区域统一调色板，避免霓虹杂乱。
- **叙事留白**：奇观不必解释来历；残迹、断裂神道、无名祭坛等保留**志怪留白**，让玩家自己脑补。
- **与动植物的统一**：后续生物与植被在**轮廓夸张度、神异配色**上与神山同一套语言，天气可强化能见度与氛围（雾、细雪、尘光等），共同完成「进域一震」的体验。

动植物与天气在第二阶段与神山奇观**同一套美学基准**一起迭代，避免先做完零散 prefab 再硬贴风格。

## 第三版计划：原创资源边界（何时必须「出图 / 出道具」）

与实现侧约定对齐，避免「口头有功能、工程无资源」：

- **第一版线（当前 ≈ 0.1.x）**：西山域 **只使用 Klei 原版已存在的 prefab** 做 `distributeprefabs` 与世界生成；**不要求**自制贴图、新道具、新 anim。辨识靠布景组合与官方地皮占位即可。
- **第二版（计划中的神山 / 动植物 / 域内天气体感）**：一旦落地 **独占实体**（非纯原版拼装），即进入 **需要 prefab + 资源** 的轨道；允许 **占位 art** 或 **复用官方 anim 的临时 prefab** 做迭代，但须在计划里标明「占位，非终版」。
- **第三版（正式资源版，发版验收）**：凡 **游戏中原本不存在**、且作为西山域卖点出现的内容，必须在第三版计划中 **逐条立项** 并验收，包括但不限于：
  - **专用地皮 / tile**：注册与 **tex、xml、小地图色** 等；
  - **神山建筑、祭坛、域内独有植被或装饰**：**anim（或等价展示方案）、贴图、音效** 与 `scripts/prefabs`；
  - **独有可拾取物 / 合成道具**：**物品栏图、地面展示、配方、`STRINGS`** 等。
- **原则**：第三版发版清单里 **不得** 把「无贴图 / 无 prefab 定义」的条目标为完成；若某功能暂不配套美术，则留在第二版「占位」阶段，不记入第三版验收。

上述需求已固化为待办 **`phase3-original-assets`**，进入第三版排期时拆分为具体美术与脚本子任务。

## 风险与对策（简短）

- **世界生成耦合**：西山域与 Klei 默认 `story`/任务池的拼接需反复用新档验证；优先最小侵入（新增 task + 挂入现有 level），避免整表替换；若采用「生成后改写地皮」方案，须单独评估与任务图拼接的边界情况，并增加选址失败时的回退策略（例如缩小域面积、换 seed 重试、或降级为仅布景标记等——具体在实现笔记中定）。
- **联机一致性**：主世界内生成由主机决定，客机无需单独生成逻辑；注意任何仅主机运行的 worldgen 钩子不要依赖客机本地状态。
- **范围蔓延**：建筑、动植物、天气容易拉长周期；本阶段严格只做区域生成稳定，内容系统后置。
- **气质跑偏**：若做成普通换皮山林、或西式浮空城/精灵塔，会偏离「山海经神山」；第二阶段用「西山经关键词 + 神山奇观草图 + 剪影测试」做评审，确保第一眼惊艳且符号锚点仍在华夏志怪一侧。
